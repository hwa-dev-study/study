# 스레드란(Thread)
- 프로세스(process)내에서 실제로 작업을 수행하는 주체를 의미한다.
- 모든 프로세스에는 한 개 이상의 스레드가 존재하여 작업을 수행한다.
- 스레드는 운영체제가 CPU 시간을 할당하는 기본 단위 이다.
- 두 개 이상의 스레드를 가지는 프로세를 멀티스레드 프로세스(multi-threaded process)라고 한다.


#### 스레드 동기화(Thread Synchronization)
- 스레드 동기화는 하나의 공용된 자원이나 변수에 여러 개의 스레드가 접근할 때, 
  스레드들이 순서를 지켜서 사용하고 다른 스레드가 사용 중일 때는 사용하지 못하게 만드는 것이다.
- 스레드를 동기화하는 방법은 lock을 사용하는 것이다. 
  스레드 락을 하기 위한 객체를 하나 만들어서 lock()을 해주면 묶인 블럭이 한 스레드에서 실행되는 동안
  같은 객체의 lock으로 묶인 스레드는 멈춘상태로 해당 코드를 진행하지 못하게 된다.
  lock()함수로 동기화하여 실행하면 겹쳐지지 않는다.
- 스레드 동기화에 단점이 있는데 동기화되는 부분은 동시에 처리가 안되고 한 스레드씩 작업을 진행하기 때문에
  프로그램 속도가 느려질 수 있다.
  또한, 스레드의 동기화 구조가 복잡한 경우라면, 데드락이 발생할 수도 있다.
- 스레드는 동시 처리를 하기에 유용한 방법이지만, 호출 순서를 보장할 수 없고, 디버깅이 어려운 구조이므로
  사용시 조심해야 한다.
